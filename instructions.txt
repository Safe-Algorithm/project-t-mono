Tourism Marketplace Platform – Spec & Scaffold
Scope

We are building a tourism marketplace platform with the following components:

    Backend (FastAPI + SQLModel)

    Web Admin Panel (Next.js + React + TS)

    Providers Panel Web Frontend (Next.js + React + TS)

⚠️ The Mobile App for tourists will be implemented later, not in scope now.
Core Concept

The app connects tourism providers (companies offering trips) with tourists (end users who book trips).
Functional Requirements
Provider Onboarding

    Providers register via the Providers Panel Web Frontend.

    They must supply:

        User credentials: email, password, name, phone.

        Company credentials: name, email, phone, documents, logo.

        Multiple documents allowed → store in JSONB.

        company_metadata = Column(
            "metadata", JSONB, nullable=True
        )  # {company_registration_file: s3_path, logo: s3_path, ...}

    Creates:

        New user (initial provider user).

        New provider request (pending approval).

Admin Approval

    Admins handle provider requests in the Web Admin Panel.

    Can:

        Approve → provider company created, initial user becomes Super Provider User.

        Deny → denial reason must be provided (visible to provider).

Roles
Provider Roles

    Super Provider User

        Invite/delete provider users.

        Edit company profile.

        Manage trips fully.

    Provider User

        Manage trips.

        Cannot invite/delete users or edit company profile.

Admin Roles

    Super Admin

        Full access: manage providers, trips, users.

        Can invite/delete admin users.

    Admin User

        Same as Super Admin, except cannot invite/delete admin users.

Trips

    Providers can create trips with:

        Images (upload → S3).

        Title, description.

        Start/end datetime.

        Pickup location (Google Maps).

        Start city/country.

        Destination cities/countries.

        Accommodation included (boolean).

    Trips include:

        Subscribers (normal users who pay).

        Reviews (from subscribed users only).

    Cancellation Rule:

        No cancellation allowed within 2 days before start.

        If canceled earlier → refund all subscribers.

Normal Users (Tourists – via Mobile App later)

    Register with email or phone.

        Email → verification via SendGrid/Mailgun (feature flag via env).

        Phone → dummy OTP for now.

    Actions:

        Browse trips, filter by: start city, destination, start date, duration.

        View provider profiles (image, name, description, list of trips).

        Join trips (pay via Checkout.com sync flow).

        Leave reviews (only for joined trips).

        Manage profile (reset password, update name/email/phone).

        Cannot leave trips after joining.

Technical Requirements
Backend (FastAPI, Python 3.12)

    Stack: FastAPI, SQLModel, PostgreSQL, Alembic, Redis, Pydantic, Poetry.

    Features:

        Pytest unit tests for all endpoints.

        Mock 3rd-party integrations:

            S3 (trip images, company docs).

            Checkout.com (payments/refunds).

            SendGrid/Mailgun (emails).

            OTP (dummy).

        Localization (English only at start, but extensible).

        JSONB storage for flexible company metadata.

Frontends (Next.js + React + TS)

    Web Admin Panel

    Providers Panel

    Features:

        Role-based access control.

        CRUD for trips, companies, users.

        Status tracking for provider requests.

        Invitation system (email-based).

Infrastructure

    Each project must be dockerized separately.

    Each should run standalone with docker compose up.

    Root-level docker-compose.yml runs all 3 together.
